🚫 Critical: Headers Must Match Cell Alignment

If cells are right-aligned, headers MUST also be right-aligned. Mismatched alignment breaks the visual column and makes scanning difficult.

  • Numbers: Right-align both header AND cells
  • Text: Left-align both header AND cells
  • Booleans: Center both header AND cells
  • Actions: Right-align both (or center both)

🚫 Common Alignment Mistakes to Avoid
  • Left-aligning numbers - Makes comparison difficult, hard to scan vertically
  • Right-aligning text - Unnatural reading flow, harder to find start of text
  • Center-aligning everything - Looks neat but reduces scannability
  • Mismatched headers and cells - Header left, cells right breaks the column
  • Inconsistent date alignment - Pick one format and stick to it
  • Forgetting tabular-nums - Variable-width digits cause misalignment
✓ What is tabular-nums?

font-variant-numeric: tabular-nums makes all digits (0-9) the same width for perfect vertical alignment in numeric columns.

Why it matters: By default, "1" is narrow and "8" is wide. Tabular-nums fixes this by giving every digit equal width (like monospace, but only for numbers).

Vertical Alignment & Text Wrapping

📐 Vertical Alignment Rules
  • Default alignment: Top (align-items: flex-start) - Content aligns to top of cell
  • Single-line cells: Center vertically for visual balance
  • Multi-line cells: Top-align to maintain scanline consistency across row
  • Mixed content rows: Always use top-align (some cells wrap, some don't)
⚠️ Text Wrapping Strategy
  • Short text (codes, IDs): Never wrap - use ellipsis (...) if needed
  • Item names: Allow 2-line wrap maximum, then ellipsis
  • Long descriptions: Truncate with ellipsis, show full text in detail view or tooltip
  • Numbers: Never wrap - always single line
  • Dates: Never wrap - use consistent format (DD/MM/YYYY)

  • Center-aligning multi-line cells - Creates uneven scanlines, hard to read across rows

Table Component Types

ℹ️ MUI v9 provides two table solutions:
  • Basic Table (MUI Table) - Simple HTML <table> element. Use for: small datasets (<100 rows), read-only displays, simple layouts, reports. No built-in column resizing.
  • DataGrid (MUI X) - Advanced component using nested <div> elements. Use for: large datasets (100+ rows), interactive features (sorting, filtering, selection, pagination), complex column types. Includes built-in column resizing - users can drag column separators.

Column Resizing

🖱️ Interactive Resizing (DataGrid Only):
MUI X DataGrid supports column resizing by default. Users can drag the right edge of any column header to adjust width. To disable for specific columns, set resizable: false in the column definition. For Basic Tables, column widths are fixed in code only.

Column Width Control

ℹ️ How to Adjust Column Widths

Column widths balance readability with space efficiency. Use fixed widths for predictable content (codes, dates, numbers) and flexible widths for variable content (names, descriptions).

Basic Table (HTML) - Adjust in the <th> tag:

// FIXED WIDTH - Use for predictable content <th style="width: 120px;">Item Code</th> // Codes: 100-150px <th style="width: 110px;">Expiry Date</th> // Dates: 100-120px <th class="numeric" style="width: 100px;">Quantity</th> // Numbers: 80-120px // FLEXIBLE WIDTH - Takes remaining space <th>Item Name</th> // No width = expands to fill available space // EXAMPLE: Making Item Name wider // Option 1: Reduce other columns (120px → 100px) // Option 2: Add min-width to prevent it getting too narrow <th style="min-width: 200px;">Item Name</th>

DataGrid (MUI X) - Set in column definitions:

// FIXED WIDTH - Exact pixel value { field: 'itemCode', headerName: 'Item Code', width: 140 // Fixed at 140px } // FLEXIBLE WIDTH - Proportional to available space { field: 'itemName', headerName: 'Item Name', flex: 1 // Takes remaining space } { field: 'supplier', headerName: 'Supplier', flex: 2 // Gets 2x space compared to flex: 1 } // MINIMUM WIDTH - Prevents column getting too narrow { field: 'reference', headerName: 'Reference', flex: 1, minWidth: 150 // Won't shrink below 150px } // DISABLE RESIZING - Prevent user from dragging to resize { field: 'status', headerName: 'Status', width: 140, resizable: false // User cannot resize this column } // CONSTRAIN RESIZING - Set min/max limits { field: 'supplier', headerName: 'Supplier', width: 180, minWidth: 120, // Can't resize smaller than 120px maxWidth: 300 // Can't resize larger than 300px }

🖱️ How Users Resize Columns in DataGrid:
Hover over the right edge of any column header - the cursor changes to a resize handle (↔). Click and drag left/right to adjust width. Release to set the new width. This works for all columns by default unless you set resizable: false.

💡 Tip: Start with these recommended widths, then adjust based on your actual data:
• Codes/IDs: 100-150px
• Dates: 100-120px
• Small numbers: 80-100px (e.g., Quantity)
• Large numbers: 110-140px (e.g., Currency with $ sign)
• Short text: 120-160px (e.g., Status)
• Long text: Use flex/no width (e.g., Item Name, Supplier)

8.5 Touch Target Sizes

⚠️ Minimum Touch Targets: 48×48px
  • Checkboxes: 20px visual size + 14px padding each side = 48×48px total
  • Icon buttons: 48×48px minimum clickable area
  • Table rows: 52px minimum height (comfortable density)
  • Tablet mode: 64px row height recommended (spacious density)

8.6 Never Rely on Color Alone

🚫 Color + Text/Icons Required

Status must never be conveyed by color alone. Always pair with:

  • Text labels ("Delivered", "In Transit")
  • Icons or status dots
  • Patterns or shapes for charts/graphs
